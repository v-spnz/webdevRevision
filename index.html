01 🌐 The World Wide Web
theory ▾

The WWW is not a simple structure — it is a complex network made up of multiple topology types simultaneously.

  • Topology types: star-shaped, grid-shaped, tree-shaped, ring-shaped, fully-connected
  • In practice the WWW is a complex combination of all the above
  • Key property: a small number of big nodes (hubs) connect to a huge number of small nodes
  • This is described by a Power Law Distribution — most pages have few links; a tiny number have enormous numbers of links
  • The graph grows by preferential attachment — new pages tend to link to already-popular pages, making hubs even larger
Power Law: The probability a node has k connections ∝ k−γ. A few hubs dominate the entire network — this is why Google, Facebook, and Wikipedia get the majority of web traffic.
02 🏗️ Web Application Architecture
architecture ▾

Monolithic

Single deployable unit. Tightly coupled. Hard to scale. Classic 3-tier: Presentation → Logic → Data.

Microservices

Loosely coupled, independently deployable. Each service owns its own data. Easier to scale.

  • Serverless: cloud provider manages servers. Pay only when code runs. Scales instantly — ideal for event-driven workloads
  • Edge Computing: code runs closer to the user or device — reduces latency
  • CI (Continuous Integration): developers push code frequently; automated tests run on each push
  • CD (Continuous Deployment): code is automatically deployed after tests pass
PHP Slim Framework can build REST microservices in a serverless-style architecture — as shown in lecture slides.

05 📦 Variables & Constants
core ▾

Variables store values in memory. The data type is determined by the value assigned to it.

  • Must begin with a $ dollar sign
  • Can contain letters (A–Z, a–z), numbers (0–9), or underscore _
  • Cannot start with a number — cannot contain spaces
  • Are case sensitive — $name and $Name are completely different variables
php — variables & data types 
$courseName   = "Web Development";  // string
$lectureHours = 2;                   // integer
$hasLabs      = TRUE;               // boolean
$avgScore     = 87.5;               // float

echo $courseName;  // Web Development

Constants hold values that do not change during execution. Defined with define().

  • Names use ALL UPPERCASE letters by convention
  • Do not begin with a $ sign
  • Value can be a string, number, or Boolean
php — constants 
define("MAX_USERS", 100);
define("SITE_NAME", "CabsOnline");
define("DEBUG_MODE", FALSE);

echo MAX_USERS;   // 100 — no $ sign needed
06 📋 Arrays
core ▾

Indexed array

Items accessed by a numeric index starting at 0. Created with array().

Associative array

Items accessed by a named string key using the => operator.

php — arrays 
// Indexed — course uses array() constructor syntax
$colours = array("red", "green", "blue");
echo $colours[0]; // red

// Associative
$student = array("name" => "Spencer", "grade" => "A");
echo $student["name"]; // Spencer

echo count($colours); // 3
  • count($arr) — returns total number of elements
  • array_merge($a, $b) — combines; duplicate keys overwritten by second array
  • $a + $b — combines; duplicate keys kept from first array
  • print_r($arr) — outputs full array structure for debugging
  • sort($arr) — ascending, resets numeric keys
  • asort($arr) — sort by value, preserves keys
  • ksort($arr) — sort by key alphabetically

08 📎 include vs require
files ▾

File not found → fatal error, script stops. Use for critical files like DB credentials.

File not found → warning only, script continues. Use for optional files.

09 🔢 Searching & Sorting Arrays
algorithms ▾

Linear Search

Checks every element one by one. Works on unsorted data. O(n) time complexity.

Binary Search

Splits search space in half each time. Requires sorted data. O(log₂ n) time complexity.

Binary search on 100 items: max 7 guesses — ceil(log₂ 100) = 7. Much faster than linear, but the array must be sorted first.

Quicksort — divides the array around a pivot, recursively sorts both halves. PHP uses this internally for sort(). Average: O(n log n). Worst case: O(n²).

php — sorting functions 
$nums = array(3, 1, 4, 1, 5);
sort($nums);    // [1, 1, 3, 4, 5] — ascending, resets keys
rsort($nums);   // [5, 4, 3, 1, 1] — descending

$ages = array("Bob"=>25, "Ana"=>19, "Zoe"=>22);
asort($ages);   // sort by value, keep original keys
ksort($ages);   // sort by key alphabetically
10 🌍 PHP Superglobals
core ▾

Superglobals are built-in PHP variables that are always accessible in any scope — inside functions, classes, or files — without needing to declare them.

SuperglobalWhat it contains
$_GETData sent via URL query string — e.g. page.php?name=Ali
$_POSTData sent via HTTP POST body — form submissions
$_REQUESTCombines GET, POST, and COOKIE data
$_SESSIONSession variables — persist across multiple pages
$_COOKIECookie values stored in the user's browser
$_SERVERServer and execution environment information
php — $_GET example (from lecture) 
<?php
  // URL: page.php?name=Spencer&pwd=pass123
  $name = $_GET["name"];
  $pwd  = $_GET["pwd"];
  echo $name . " : " . $pwd;
?>
GET vs POST: Use $_GET when data can be public (visible in URL). Use $_POST for sensitive data like passwords — sent in the request body, not the URL.

13 🔐 Sessions & State Management

decisions ▾

14 🔁 switch Statement
decisions ▾
  • Use switch when testing one variable against many possible values
  • Each case must end with break; to prevent fall-through to the next case
  • default: runs if no case matches — equivalent to a final else
php — switch statement 
switch ($day) {
    case "Mon":
        echo "Monday";
        break;
    case "Fri":
        echo "Friday";
        break;
    default:
        echo "Other day";
}

16 🔌 Web Services & REST
architecture ▾

Traditional (SOAP)

XML-based. Strict contracts via WSDL. Heavyweight. Described with .wsdl files.

REST

HTTP methods. Lightweight. JSON-based. Resource-oriented URLs. Described via OpenAPI (YAML).

  • REST uses: GET (read), POST (create), PUT (update), DELETE (delete)
  • WSDL = Web Services Description Language — describes a SOAP service in XML
  • OpenAPI (formerly Swagger) — describes REST APIs in YAML format
php — url routing (from lecture) 
// http://localhost/api/v1/employees → getEmployees() via GET
$app->get('/employees',       'getEmployees');
$app->get('/employee/{id}',  'getEmployee');
$app->post('/create',        'addEmployee');
$app->put('/update/{id}',    'updateEmployee');
$app->delete('/delete/{id}', 'deleteEmployee');
17 📄 XML & JSON
data formats ▾

Why use XML and JSON? — Simplicity, extensibility, interoperability, platform-independent, industry-accepted, supports Unicode, one data source — multiple views.

XML Documents must be:

  • Well-formed (compulsory) — correct syntax, all tags properly closed and nested
  • Valid against a Schema/DTD (optional) — validates structure and data types
  • Parsers check for well-formedness. Validating parsers also check against DTD/Schema.
  • A character reference uses decimal/hex to represent special characters (e.g. &#169; = ©)

JSON (JavaScript Object Notation) — lightweight, human-readable. The mainstream format for web data transfer in Web 2.0.

json — data structure 
{
  "name": "Spencer",
  "age": 21,
  "courses": ["COMP721", "COMP712"],
  "active": true
}

JSON Schema — defines the expected structure. type is the only mandatory constraint.

json — schema (from lecture) 
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "Product",
  "description": "A product in the catalog",
  "type": "object"
}
18 ⚡ Web 2.0 & the Fetch API
ajax ▾

Web 2.0 resolves limitations of the classic request-response model. The goal: allow dynamic change in the interface whilst the user keeps interacting — without full page reloads.

  • JSON — mainstream data transfer format between client and server
  • DOM — dynamically manipulate page objects (document.getElementById, innerHTML)
  • Fetch API — makes asynchronous HTTP requests (modern replacement for XHR)
  • Enables: partial page updating, invisible data retrieval, live feeds, drag-and-drop, dynamic menus
javascript — fetch api (course pattern) 
fetch("booking.php?action=getAll")
  .then(response => response.json())
  .then(data => {
    data.forEach(row => {
      document.getElementById("results")
        .innerHTML += "<tr><td>" + row.name + "</td></tr>";
    });
  });
XHR vs Fetch: XHR (XMLHttpRequest) was the original AJAX approach. This course uses the modern Fetch API — promise-based, cleaner syntax.
19 🗄️ PHP & MySQL (mysqli)
database ▾
  • Always use require_once to load DB credentials — never hardcode them
  • Use procedural mysqli style as taught in lectures — not PDO or OOP mysqli
  • mysqli_connect($host, $user, $pass, $db) — opens a connection
  • mysqli_query($conn, $sql) — executes a SQL statement
  • mysqli_fetch_assoc($result) — fetches one row as an associative array
  • mysqli_num_rows($result) — returns number of rows in the result set
  • mysqli_close($conn) — closes the connection
  • json_encode($array) — converts PHP array to JSON string for Fetch response
php — full mysqli pattern (course standard) 
require_once('../../sqlinfo.inc.php');

$conn   = mysqli_connect($sql_host, $sql_user, $sql_pass, $sql_db);
$sql    = "SELECT * FROM bookings";
$result = mysqli_query($conn, $sql);

$rows = array();
while ($row = mysqli_fetch_assoc($result)) {
    $rows[] = $row;
}
echo json_encode($rows);
mysqli_close($conn);
TRUNCATE vs DELETE: TRUNCATE removes all rows and resets AUTO_INCREMENT to 1. DELETE removes rows but does not reset AUTO_INCREMENT.
